# 後工程 詳細

このページは、スライド本編「後工程（第3部）」を補足する。後工程は、前工程で回路パターンを
形成したウェハーを個々のチップに切り出し、外部と電気的に接続し、外部環境から保護する一連の
工程。近年は先端パッケージング技術の重要性が急速に高まっており、性能向上の鍵を握る分野と
して扱う。

## 後工程全体フロー

```{mermaid}
:caption: 後工程の流れ
:align: center

flowchart LR
  A["裏面研削"] --> B["ダイシング"] --> C["ボンディング"] --> D["封止"] --> E["検査"]
```

前工程が「回路パターンをウェハー上に形成する」工程であるのに対し、後工程は「完成した
ウェハーを個々の製品として仕上げる」工程にあたる。

## バックグラインド・ダイシング

ウェハーの裏面を薄く削る「バックグラインド」と、個々のチップに切り分ける「ダイシング」が
後工程の入り口。積層パッケージやスマートフォン向けの薄型化が進むほど、ウェハーをより薄く
精密に加工する技術の重要性が増す。

| 企業 | 装置 | 位置づけ |
| --- | --- | --- |
| ディスコ | バックグラインダー / ダイシングソー | 両分野で世界的な大手 |
| 東京精密（Accretech） | バックグラインダー / ダイシングソー | ディスコと並ぶ主要プレイヤー |

後述の3D実装・積層化が進むほど、ダイをより薄く均一に加工する技術が製品の歩留まりを左右する
ため、この工程の重要性は今後さらに増していくと見られる。

## ボンディング・封止＋装置

切り出したチップを基板やリードフレームに固定し、電気的に接続する工程。ワイヤボンディング・
ダイボンディング・フリップチップボンディングなど、用途に応じて接続方式を使い分け、その後
モールド樹脂で封止して外部環境から保護する。

| 企業 | 装置 | 位置づけ |
| --- | --- | --- |
| ASMPT | ワイヤ/ダイボンダー | ボンディング装置全般で大手 |
| Kulicke & Soffa | ワイヤボンダー | ワイヤボンディングで主要プレイヤー |
| Besi | ダイ/フリップチップボンダー | 高精度実装装置で大手 |
| 新川 | ワイヤボンダー | 日本の主要プレイヤー |

## パッケージ基板とABF

高性能チップの周辺には、微細配線を持つパッケージ基板が必要になる。フリップチップBGA
（FC-BGA）基板の層間絶縁材料として使われる**ABF（Ajinomoto Build-up Film）**は、
味の素がほぼ独占的に供給しており、本編の中でも特に確度の高い「日本の強み」の代表例。

| 分野 | 代表企業 | 位置づけ |
| --- | --- | --- |
| ABF（絶縁材料） | 味の素 | PC向けほぼ100%、ハイエンド帯でも95〜100%（確度高） |
| FC-BGA基板 | Ibiden / 新光電気工業 / Unimicron / AT&S | 主要な基板メーカー各社 |
| ガラスコア基板 | Intel | 次世代基板として開発を推進 |

ABFは、もともと味の素が化学品事業として開発した樹脂フィルムで、多層配線を可能にする
絶縁材料として先端ロジック向けFC-BGA基板の事実上の標準になっている。この独占的な地位は
複数のソースで一致しており、本編を通じても確度の高い数値の一つとして扱う。なお、ABFの
供給先となる個別の顧客企業名（どのファブレス・ファウンドリ向けか）については、本ドキュメント
では特定しない。

次世代の基板技術としては、樹脂ではなくガラスをコア材に使う「ガラスコア基板」の開発を
Intelが推進している。配線の微細化・平坦性・熱膨張特性の面で樹脂コアの限界を超える技術
として注目されている。

## 先端パッケージング

複数のチップを1つのパッケージに統合する2.5D/3D実装やチップレット構成、HBM（広帯域メモリ）
の搭載が、先端半導体の性能を左右するようになっている。大手各社はそれぞれ独自ブランドの
先端パッケージング技術を展開しており、ブランド名の対応関係を押さえておくと業界動向を
追いやすい。

| 企業 | ブランド | 概要 |
| --- | --- | --- |
| TSMC | CoWoS / InFO / SoIC | 2.5D実装（CoWoS）・ファンアウト型実装（InFO）・3D積層（SoIC） |
| Intel | Foveros / EMIB | 3D積層（Foveros）・チップ間ブリッジ接続（EMIB） |
| Samsung | I-Cube / X-Cube | 2.5D実装（I-Cube）・3D積層（X-Cube） |

CoWoS（Chip-on-Wafer-on-Substrate）はTSMCの2.5Dパッケージング技術で、ロジックダイと
HBMスタックをシリコンインターポーザ上に並べて実装する方式。AI半導体の高性能化を支える
主要技術の一つとして、CoWoSの生産能力自体が業界の供給制約要因になることもある。

異なるベンダーのチップレットを組み合わせるには、接続方式の標準化が欠かせない。
**UCIe（Universal Chiplet Interconnect Express）**はその業界標準規格で、AMD・Arm・ASE・
Google・Intel・Meta・Microsoft・Qualcomm・Samsung・TSMCなどが創設メンバーとして参加する。
UCIeが普及すれば、異なる企業が製造したダイを1つのパッケージ内で組み合わせる「ミックス&
マッチ」型の設計がより実用的になる。ISAレベルの標準化（RISC-V）と合わせて、設計エコシステム
のオープン化を象徴する動きと言える（ISA側の詳細は {doc}`design` ページ参照）。

## HBM・積層化・3D実装

チップを水平に並べるだけでなく、垂直に積み重ねて配線距離を縮める3D実装が進んでいる。
TSV（シリコン貫通電極）でチップ間を垂直に接続する手法に加え、はんだバンプを介さずに
直接接合する**ハイブリッドボンディング**や、ウェハー同士を貼り合わせる
**Wafer-on-Wafer（WoW）**が採用され始めている。HBM（広帯域メモリ）は、この積層技術を
象徴する製品であり、DRAMダイを複数枚積層してTSVで接続し、ベースとなるロジックダイの上に
搭載する構造を取る。

```{mermaid}
:caption: HBMの積層構造（概念図）
:align: center

flowchart TD
  D4["DRAMダイ #4"] -->|TSV| D3["DRAMダイ #3"] -->|TSV| D2["DRAMダイ #2"] -->|TSV| D1["DRAMダイ #1"] -->|TSV| BASE["ベースダイ/ロジック"]
```

生成AI向けの演算需要が急増する中、HBMは帯域幅の制約を解消する要となっており、積層段数の
増加（8層・12層・16層など）と歩留まり確保の両立が技術的な焦点になっている。HBM市場の
企業順位（SK hynixが首位）は {doc}`industry` ページ「メモリ」で扱う。

## テスト工程とOSAT

封止後のチップは、テスタを用いた電気的検査で良品を選別する。この最終検査を含む後工程全体
を専門に請け負う企業群が**OSAT（Outsourced Semiconductor Assembly and Test）**と呼ばれる。

| 分野 | 代表企業 | 位置づけ |
| --- | --- | --- |
| 半導体テスタ | Advantest / Teradyne | テスタ市場の二強 |
| OSAT | ASE（首位）/ Amkor / JCET | 後工程を専門に請け負う受託企業（概況） |

OSATは、ファウンドリが前工程に専業化したのと同様の理由で、後工程に専業化した企業群。
先端パッケージングの重要性が増すにつれ、OSAT各社も自社ブランドの先端パッケージング技術
（TSMCのCoWoS等に対抗する独自技術）への投資を強めている。

## 後工程まとめ

裏面研削・ダイシングからボンディング・封止・パッケージ基板・先端パッケージング・テストまで、
後工程には多くの専業企業が関わる。特に封止材・リードフレーム・ABFなどの材料分野では、
日本企業の存在感が大きい。

| 分野 | 代表企業 | 位置づけ |
| --- | --- | --- |
| 加工装置 | ディスコ / 東京精密 / ASMPT / Besi / Kulicke & Soffa / 新川 | バックグラインド・ダイシング・ボンディングを担う |
| 材料 | 住友ベークライト / ナミックス / 田中電子工業 / 日鉄マイクロメタル / 三井ハイテック / 味の素 | 封止材・リードフレーム・ABFで日本勢が強み |
| パッケージング・テスト | TSMC / Intel / Samsung / Advantest / ASE | 先端パッケージングとテスト・OSATの主要プレイヤー |

前工程が装置・材料の両面で日本企業の存在感が大きい部だったのに対し、後工程は**材料分野に
日本企業の強みが集中**し、先端パッケージング自体の主導権はTSMC・Intel・Samsungといった
ファウンドリ・IDM側が握るという構図が特徴的である。

数値の出典・確度区分は {doc}`sources` ページに、専門用語の解説は {doc}`glossary` ページに
集約している。
