# 設計

このページは、スライド本編「設計（第1部）」を補足する。産業の分業構造、設計工程の流れ、
設計エコシステムを支える {term}`EDA`・IP・命令セット（{term}`ISA`）、そして主要{term}`ファブレス`企業について、
散文と表で詳しく解説する。全体の位置づけは {doc}`overview` ページの「分業構造」節も参照。

## 産業の分業構造（IDM／ファブレス／ファウンドリ／OSAT）

半導体産業の事業モデルは大きく2つに分かれる。

```{mermaid}
:caption: IDMと水平分業型
:align: center

flowchart LR
  subgraph IDM["IDM（垂直統合型）"]
    I1["設計"] --> I2["製造"]
  end
  subgraph SPLIT["水平分業型"]
    F["ファブレス：設計に特化"] -->|製造委託| FD["ファウンドリ：製造専業"]
    FD --> O["OSAT：組立・検査専業"]
  end
```

**{term}`IDM`（Integrated Device Manufacturer）** は、設計から前工程・後工程まで自社の工場で
一貫して行う企業。Intel やローム、メモリ事業を含む Samsung が代表例。自社工場（ファブ）
への巨額の設備投資を継続できる体力が前提になる。

**水平分業型** では、設計に特化する「ファブレス」企業が、製造を専業の「{term}`ファウンドリ`」に
委託する。後工程（組立・検査）はさらに専業の「{term}`OSAT`」が請け負う。ファブレスは工場を持たない
ぶん身軽で、設計・アーキテクチャの差別化に投資を集中できる。ファウンドリは複数の顧客の
需要を束ねることで、装置・材料への巨額投資を回収する規模を確保できる。

先端ロジックの微細化が進むほど、1世代の製造装置・材料への投資額が跳ね上がる。この結果、
自社ファブを維持できる企業は限られ、TSMC のような巨大ファウンドリへ製造が集約される流れが
強まっている。Apple のように自社で最終製品向けSoCを設計しながら、製造は TSMC に全面委託
するファブレス企業が典型例。

## ファブレスとファウンドリの関係

ファブレスとファウンドリの関係は、単なる「発注・受注」にとどまらない。最先端{term}`ノード`では
量産開始のかなり前から、ファブレス側の設計データとファウンドリ側の製造プロセス（デザイン
ルール、{term}`PDK`: Process Design Kit）をすり合わせる**協調設計（{term}`DTCO`: Design-Technology
Co-Optimization）**が不可欠になっている。歩留まりを確保するには、回路設計の段階から
製造上の制約を織り込む必要があるためだ。

この関係性は、ファウンドリを選ぶこと自体が経営判断になることも意味する。最先端ノードを
提供できるファウンドリは TSMC・Samsung・Intel の少数に限られており（詳細は {doc}`industry`
ページ「ファウンドリ勢力図」参照）、ファブレス企業は性能・コスト・供給安定性・地政学リスク
を勘案してファウンドリを選定する。1社の技術に依存しすぎないよう複数ファウンドリを併用する
「マルチソース化」も、大手ファブレスの間で意識されている論点。

## 設計工程の流れ

チップ設計は、抽象度の高い仕様から、より具体的な物理レイアウトへと段階的に詳細化していく
流れで進む。

```{mermaid}
:caption: 設計工程の流れ
:align: center

flowchart LR
  S["仕様"] --> L["論理設計"] --> C["回路設計"] --> P["レイアウト"] --> V["検証"]
```

| 段階 | 内容 |
| --- | --- |
| 仕様 | チップに求める性能・機能・消費電力などの要件を定義する |
| 論理設計 | 仕様をハードウェア記述言語（HDL）で回路の論理動作として表現する |
| 回路設計 | 論理設計をトランジスタレベルの回路構成に落とし込む |
| レイアウト | 回路を実際のシリコン上の配置・配線パターンに変換する（フロアプラン・配置配線） |
| 検証 | 設計が意図通り動作するか、製造可能か（DRC/LVS等）をシミュレーションで確認する |

検証工程は製造に進む前の最後の関門であり、大規模なチップほど検証にかかる計算資源・期間が
急増する。この負荷の増大が EDA ツールの高度化を後押ししている。

## EDA・IP・ISA：設計エコシステムの土台

設計工程を実際に支えるのは、ソフトウェア（EDA）・再利用可能な回路資産（IP）・命令セット
アーキテクチャ（ISA）という3層のエコシステムである。

| 領域 | 代表企業・規格 | 位置づけ |
| --- | --- | --- |
| EDA（設計自動化ツール） | Synopsys / Cadence / Siemens EDA | 論理設計からレイアウト・検証まで、設計工程全体を支えるソフトウェアの大手3社 |
| IP（回路設計資産） | Arm | モバイル・組込み向けの命令セット・コア設計のライセンスで主流 |
| オープンISA | {term}`RISC-V`（Tenstorrent 等が採用） | ライセンス不要のオープンな命令セット。Arm に代わる選択肢として採用が広がる潮流 |

**EDA** は、数十億トランジスタ規模の回路を人手だけで設計・検証することが不可能なため、
論理合成・配置配線・タイミング解析・検証を自動化するソフトウェア群を指す。Synopsys・
Cadence・Siemens EDA（旧 Mentor Graphics）の3社が事実上の標準ツールを提供しており、
最先端プロセスの設計には各社ツールへの習熟がほぼ前提になる。

**IP** は、CPU コアやインターフェース回路など、ゼロから設計せずライセンスで組み込める
回路資産を指す。Arm はモバイル・組込み向け命令セット（ISA）とコア設計の両方をライセンス
提供する事業モデルで、スマートフォン向け SoC のほぼすべてに採用されている。

**RISC-V** は、Arm のようなライセンス費用が発生しないオープンソースの ISA。命令セット
仕様が公開されているため、誰でも自由にプロセッサを設計・実装できる。Arm 一強だった
CPU アーキテクチャの選択肢を広げる潮流として、Tenstorrent をはじめ採用企業が増えている。
日本のファウンドリ Rapidus は、この RISC-V エコシステムの担い手である Tenstorrent と
2nmロジックの IP 共同開発で提携している（提携の性格については {doc}`industry` ページの
「ファウンドリ勢力図」で詳述する。**製造委託契約ではなく IP 共同開発提携**である点に
注意が必要）。

```{note}
{term}`チップレット`（複数のダイを1パッケージに統合する構成）が普及するにつれ、ISA だけでなく
**ダイ間の接続方式の標準化**も設計エコシステムの重要な論点になっている。その業界標準
{term}`UCIe`（Universal Chiplet Interconnect Express）は AMD・Arm・Intel・TSMC・Samsung 等が
創設メンバーとして参加する規格で、異なるベンダーのダイを組み合わせる際の相互接続性を
担保する。UCIe はパッケージング段階の技術であるため、詳細は {doc}`backend` ページ「先端
パッケージング」で扱う。
```

## 主要ファブレス

先端ロジックの設計を担う代表的なファブレス企業。

| 企業 | 主な注力領域 | 備考 |
| --- | --- | --- |
| NVIDIA | GPU・AIアクセラレータ | 生成AI向け需要で急成長 |
| Qualcomm | モバイル向けSoC・通信 | スマートフォン向けチップで大手 |
| AMD | CPU・GPU | サーバー・PC向けで存在感 |
| Apple | 自社製品向けSoC | 垂直統合の設計戦略（製造はTSMCに委託） |
| Broadcom | 通信・ネットワーク半導体 | カスタムASICも展開 |

これらのファブレス企業は、それぞれ異なる市場（データセンターAI、モバイル、PC/サーバー、
自社製品、通信インフラ）に特化しつつ、製造はいずれも外部ファウンドリに依存している。
この依存関係が、{doc}`industry` ページで扱うファウンドリ勢力図の重要性を裏付けている。

数値の出典・確度区分は {doc}`sources` ページに、専門用語の解説は {doc}`glossary` ページに
集約している。
